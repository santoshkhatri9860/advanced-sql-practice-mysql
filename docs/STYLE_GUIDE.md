# Style Guide

This repo is designed to be consistent and beginner-friendly.

## Exercises
- Prompts only (no answers)
- Clear objective and constraints
- Consistent formatting and naming

## Solutions
- Comment blocks explaining intent
- Prefer readability over clever tricks
- One solution per prompt when possible
- Use MySQL 8 compatible syntax

## SQL formatting
- Use uppercase for SQL keywords (SELECT, FROM, WHERE)
- Use snake_case for aliases and derived columns
- Keep lines short and readable where possible
